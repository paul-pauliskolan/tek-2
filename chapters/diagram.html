Vanliga Diagram inom IT

Här är tre vanliga sätt att visuellt beskriva hur system fungerar, hur data lagras och hur kod interagerar. Allt uppritat med Mermaid.js.

1. Systemdiagram (Arkitektur)

Ett systemdiagram visar de stora byggstenarna i en applikation och hur de kommunicerar med varandra. Det är perfekt för att få en överblick över infrastrukturen (t.ex. frontend, backend, databaser och externa tjänster).

Användningsområde: När du ska förklara för en kund eller nya utvecklare hur systemet är uppbyggt "från ovan". Cylindern representerar traditionellt en databas.

2. Databasschema (ER-diagram)

Ett Entity-Relationship (ER) diagram visar hur data är strukturerad i en databas. Det visar tabellerna (entiteterna), vilka kolumner de har, och hur de är kopplade till varandra (t.ex. "en kund kan ha flera ordrar").

erDiagram %% Definierar relationerna %% KUND ||--o{ ORDER : "placerar" ORDER ||--|{ PRODUKT : "innehaller" %% Definierar tabellerna och deras kolumner %% KUND { int kund_id PK string namn string epost } ORDER { int order_id PK int kund_id FK date skapad_datum float total_pris } PRODUKT { int produkt_id PK string produktnamn float pris }
Syntax-förklaring:
  • PK står för Primary Key (unik identifikation).
  • FK står för Foreign Key (länk till en annan tabell).
  • Symbolen ||--o{ betyder "En-till-många" (En kund kan ha noll eller flera ordrar).

3. UML-diagram (Klassdiagram)

UML (Unified Modeling Language) består egentligen av flera olika diagramtyper. När man pratar om UML inom objektorienterad programmering menar man dock oftast Klassdiagram. Det visar systemets klasser, deras attribut (variabler), metoder (funktioner) och hur de relaterar till varandra (t.ex. genom arv).

classDiagram %% Relationer mellan klasser %% Anvandare <|-- Kund : Arv (Inheritance) Kund "1" --> "0..*" Order : Placerar (Association) %% Klassdefinitioner %% class Anvandare { -String namn -String epost +loggaIn() bool +loggaUt() void } class Kund { -String leveransAdress -String betalningsMetod +laggTillOrder(Order o) void } class Order { -int orderId -float totalPris -Date datum +beraknaMoms() float +skickaKvitto() void }
Syntax-förklaring för Klassdiagram:
  • - (Minus) betyder Private (endast klassen själv kan komma åt variabeln/metoden).
  • + (Plus) betyder Public (alla klasser kan komma åt den).
  • Den obrutna pilen med vit triangel (<|--) betyder Arv / Inheritance (En Kund är en Användare).
  • Den vanliga pilen (-->) betyder Association (En Kund har Ordrar). Siffrorna 1 till 0..* betyder att "en kund kan ha noll till obegränsat antal ordrar".
